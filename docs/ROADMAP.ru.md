## v0.7 — Анонимизация лиц в видео

> Расширяет пайплайн v0.6 Hide Faces на видео. Те же 4 эффекта,
> те же 5 детекторов, но покадрово с трекингом лиц и перекодированием в видеофайл.
> **Весь пайплайн в WebWorker** — главный поток только UI.

### Архитектура

```
MP4/WebM → MP4Box demux → WebCodecs VideoDecoder → VideoFrame
  → Детекция (ключевые кадры) → ByteTrack → Эффект → WebCodecs VideoEncoder
  → mp4-muxer remux → Выходной MP4/WebM (с аудио passthrough)
```

Zero-copy где возможно: `VideoFrame` / `ImageBitmap` между стадиями, `OffscreenCanvas` для рендера, без `getImageData()` в горячем пути.

| Слой | Модуль | Назначение |
|------|--------|------------|
| Демукс | `MP4Box.js` | Разделение MP4 на видео- и аудиодорожки (`EncodedVideoChunk`) |
| Декодинг | `WebCodecs VideoDecoder` | GPU-ускоренный покадровый доступ → `VideoFrame` |
| Детекция | переисп. `anonymize.ts` | Полная детекция на ключевых кадрах; BlazeFace — дешёвый верификатор на каждом |
| Трекинг | `faceTracker.ts` | ByteTrack (двухэтапный IoU-матчинг) + адаптивный интервал ключевых кадров |
| Re-ID | `faceReid.ts` (ArcFace ONNX) | Переидентификация лиц после длинных окклюзий (v0.7.4, опционально) |
| Эффект | переисп. `anonymizeEffects.ts` | Размытие / Пикселизация / Заливка / Эмодзи на лицо на кадр |
| Кодирование | `WebCodecs VideoEncoder` | H.264 baseline основной, VP9 опционально. `isConfigSupported()` для проверки |
| Мукс | `mp4-muxer` | Объединение видео + аудио passthrough → MP4/WebM |
| Аудио | `AudioEncoder` (если перекодирование нужно) | Remux оригинального аудио без перекодирования при совпадении контейнера |
| Состояние | `videoAnonymizeStore.ts` | Боксы по кадрам, ID треков, прогресс обработки |
| Воркер | `videoWorker.ts` | Весь пайплайн (decode→track→effect→encode), `postMessage` + transferable `VideoFrame` |

### Почему не `<video>` + Canvas

- `<video>.currentTime` — асинхронный, неточный (snap к ближайшему keyframe), теряются кадры на длинных видео
- `requestVideoFrameCallback` не может декодировать быстрее реального времени — видео 30с занимает ≥30с
- `drawImage(video) → getImageData()` — GPU→CPU readback, гигантский bottleneck
- **Исправление:** WebCodecs `VideoDecoder` — точный покадровый доступ, быстрее реального времени, GPU-native

### Стратегия кодирования

| Метод | Плюсы | Минусы |
|-------|-------|--------|
| **WebCodecs H.264** | GPU-ускорение, универсален (Chrome/Edge/Safari) | Firefox ограниченно |
| **WebCodecs VP9** | Открытый кодек, хорошее сжатие | Поддержка энкодера не везде |
| **ffmpeg.wasm** | Кроссплатформенный fallback | 30+ МБ WASM, CPU, <2% пользователей |

- **Основной:** H.264 baseline через WebCodecs (Chrome, Edge, Safari 17+)
- **Fallback:** ffmpeg.wasm для очень старых браузеров (<2%)
- **Детекция:** `VideoEncoder.isConfigSupported({ codec: 'avc1.42001E' })` — не browser sniffing
- **Контейнер:** MP4 для H.264, WebM для VP9

### Трекинг лиц — ByteTrack (v0.7.1)

- **ByteTrack**: трёхэтапная ассоциация — Stage 1 (high-conf × strict cost), Stage 1.5 (unmatched high-conf rescue с адаптивным порогом), Stage 2 (low-conf × wide window). ~250 строк, без ML
- **Интервал ключевых кадров**: адаптивный — 15→60 кадров когда треки уверенные, 5 кадров когда нестабильны
- **Kalman-фильтр**: 8 состояний (x, y, w, h, dx, dy, dw, dh) + якорные позиции (ax, ay, aw, ah) для точного расчёта per-frame скорости. dw/dh предсказывают изменение размера между keyframe'ами
- **Скорость**: вычисляется как `(newPos − anchor) / elapsed` — где anchor = позиция на момент последнего update, elapsed = кадров с тех пор. Исключает накопление ошибки от dt·detectionInterval
- **Re-association**: при создании нового трека проверка рядом потерянных (lost≤5, cost<2.5) — переиспользуется старый ID с сохранением вектора скорости
- **Cost-функция**: 1−IoU для пересекающихся боксов, 1+centerDist/diag для разъехавшихся. Адаптивный порог Stage 1.5: `1.1 + lostFrames*0.06` — шире окно для давно потерянных треков
- **Затухание скорости**: только на truly lost треках (lostCount>0), не на обычных predict-кадрах. При lost: VEL_DECAY_POS=0.95, VEL_DECAY_SIZE=0.9 per frame
- **EMA-сглаживание**: α=0.55 на update, α=1.0 на predict (без лага, Kalman уже сглажен через velocity EMA)
- **Temporal mask smoothing**: low-pass фильтр (EMA α≈0.55) на smoothBox перед rasterize — убирает «мигание» маски

### Scale-Invariant эффекты для видео

- **Ядро пикселизации / радиус размытия**: привязано к размеру bbox (`kernel = bbox.width * 0.1`), а не фиксированные пиксели — без визуального «дыхания» при приближении/удалении лица
- **Feather**: пропорционально размеру bbox, не абсолютные пиксели
- **Размер эмодзи**: `bbox.width * 0.6` — адаптация к размеру лица на каждом кадре

### Аудио passthrough (v0.7.2)

- **Демукс:** `mediabunny` разбирает MP4/WebM на видео- и аудио-дорожки
- **Ремукс:** `mediabunny` (npm, ~400KB) — аудио passthrough без перекодирования
- **Нюанс:** контейнер должен совпадать (MP4→MP4 для AAC, WebM→WebM для Opus/Vorbis). При смене контейнера (MP4→WebM) — перекодирование аудио через `AudioEncoder`
- **Нет аудиодорожки?** Пропускаем, выходное видео без звука

### Память и backpressure (реализовано)

- **`VideoFrame.close()`** после каждого кадра + **`VideoSample.close()`** в try/finally — без утечек GPU-памяти
- **Streaming pipeline:** decode → track → effect → encode — один кадр в полёте, буфер EncodedVideoChunk для muxing
- **Encoder flush:** `encoder.flush()` после всех кадров перед finalize
- **Отмена:** `AbortController` + проверка `signal.aborted` в циклах

### UI

1. Загрузка видео → первый кадр + длительность/разрешение/метаданные
2. Детекция на кадре 0 → оверлей (переисп. FaceOverlay)
3. Настройка эффектов (переисп. панель эффектов AnonymizeWizard)
4. «Обработать» → прогресс: кадров готово / всего, ETA (EMA за последние 30 кадров), отмена
5. Превью: рендер каждого N-го готового кадра в `OffscreenCanvas` → отправка в главный поток (не realtime)
6. «Скачать» → выходное видео (то же разрешение/fps, аудио passthrough)
7. До/После: сравнение кадров или два видеоплеера рядом
