## v0.7 — Анонимизация лиц в видео

> Расширяет пайплайн v0.6 Hide Faces на видео. Те же 4 эффекта,
> те же 4 детектора, но покадрово с трекингом лиц и перекодированием в MP4.

### Архитектура

```
MP4/WebM → mediabunny demux → VideoSampleSink (декодированные кадры)
  → Canvas API → Детекция лиц (keyframes) → ByteTrack → Эффект → Canvas
  → VideoFrame → WebCodecs VideoEncoder (H.264) → EncodedVideoPacketSource
  → mediabunny Output (MP4) + EncodedAudioPacketSource (аудио passthrough)
```

| Слой | Модуль | Назначение |
|------|--------|------------|
| Демукс | `mediabunny Input` | Разделение MP4/WebM на видео- и аудиодорожки |
| Декодинг | `mediabunny VideoSampleSink` | Поток `VideoSample` через WebCodecs |
| Детекция | переисп. `anonymize.ts` ↔ `detectFaces()` | SCRFD-500M/10G на ключевых кадрах (640×640 с тайлингом) |
| Pose | `poseEstimate.ts` (YOLOv8-pose) | 17 ключевых точек COCO для трекинга тела — оценка позиции лица при окклюзии (опционально, v0.7.3) |
| Трекинг | `faceTracker.ts` | ByteTrack (трёхэтапный IoU-матчинг) + Kalman (8 состояний + якоря). Настраиваемый `maxLost` |
| Re-ID | `faceReid.ts` (ArcFace ONNX) | Переидентификация лиц после длинных окклюзий (v0.7.5, опционально) |
| Эффект | переисп. `anonymizeEffects.ts` (canvas) | Размытие / Пикселизация / Заливка / Эмодзи на лицо на кадр, отрисовка в canvas |
| Кодирование | `WebCodecs VideoEncoder` | H.264 (`avc1.420028`), VP9 fallback. `isConfigSupported()` для проверки |
| Мукс | `mediabunny` (`Mp4OutputFormat` + `BufferTarget`) | Объединение видео + аудио passthrough → MP4 |
| Аудио | `mediabunny EncodedAudioPacketSource` | Remux оригинального аудио без перекодирования |
| Состояние | `videoAnonymizeStore.ts` | Файл, эффекты, качество, трекинг тела, прогресс, ETA, выходной blob |
| Инференс | `inferenceClient.ts` (общий singleton-воркер) | Централизованное управление ORT-сессиями, FIFO-сериализация для безопасности WebGPU run |

**Пайплайн работает в главном потоке** (не в WebWorker). Canvas API + `VideoFrame` в горячем цикле. Mediabunny для demux/mux.

### Режимы качества

| Режим | Интервал детекции | Интервал pose | Скорость | Пропуски лиц |
|-------|-------------------|---------------|----------|--------------|
| **Быстро** (по умолчанию) | адаптивный 5–60 кадров | каждый face-keyframe | 3–10× realtime | ≤2с без маски на новом лице |
| **Качественно** | каждый кадр (1) | каждый 5-й face-keyframe | 1–3× realtime | отсутствуют |

Оба режима поддерживают **Отслеживание тела** — YOLOv8-pose запускается параллельно с face detection на keyframe'ах (троттлинг до каждого 5-го face-keyframe в Accurate-режиме, чтобы не гонять 82 МБ модель 30 раз в секунду).

### Почему не `<video>` + Canvas

- `<video>.currentTime` — асинхронный, неточный (snap к ближайшему keyframe), теряются кадры на длинных видео
- `requestVideoFrameCallback` не может декодировать быстрее реального времени — видео 30с занимает ≥30с
- `drawImage(video) → getImageData()` — GPU→CPU readback, гигантский bottleneck
- **Исправление:** mediabunny `VideoSampleSink` — WebCodecs-декодинг, точный покадровый доступ, быстрее реального времени

### Стратегия кодирования

| Метод | Плюсы | Минусы |
|-------|-------|--------|
| **WebCodecs H.264** | GPU-ускорение, универсален (Chrome/Edge/Safari) | Firefox ограниченно |
| **WebCodecs VP9** | Открытый кодек, хорошее сжатие | Поддержка энкодера не везде |
| **MediaRecorder** | Кроссплатформенный fallback | Только WebM, без аудио, медленнее |

- **Основной:** H.264 через WebCodecs → MP4 (Chrome, Edge, Safari 17+)
- **Fallback:** MediaRecorder VP9 → WebM (без аудио passthrough)
- **Детекция:** `VideoEncoder.isConfigSupported({ codec: 'avc1.420028' })` — не browser sniffing

### Трекинг лиц — ByteTrack (v0.7.1)

- **ByteTrack**: трёхэтапная ассоциация — Stage 1 (high-conf × strict cost 0.65), Stage 1.5 (unmatched high-conf rescue с per-track адаптивным порогом `1.1 + lostFrames×0.06`), Stage 2 (low-conf × wide window 1.5). ~300 строк, без ML
- **Интервал ключевых кадров**: адаптивный — 15→60 кадров когда треки уверенные, 5→30 кадров когда нестабильны. Accurate-режим фиксирует на 1.
- **Kalman-фильтр**: 8 состояний (x, y, w, h, dx, dy, dw, dh) + якорные позиции (ax, ay, aw, ah, dt) для точного расчёта per-frame скорости. dw/dh предсказывают изменение размера между keyframe'ами
- **Скорость**: вычисляется как `(newPos − anchor) / elapsed` — где anchor = позиция на момент последнего `kalmanUpdate`, elapsed = кадров с тех пор. Исключает накопление ошибки от dt·detectionInterval
- **Re-association**: при создании нового трека проверка рядом потерянных (lost≤5, cost<2.5) — переиспользуется старый ID с сохранением вектора скорости, сбрасываются smoothBoxes для избежания скачка
- **Cost-функция**: 1−IoU для пересекающихся боксов, 1+centerDist/diag для разъехавшихся
- **Затухание скорости**: только на truly lost треках (lostCount>0), не на обычных predict-кадрах. При lost: VEL_DECAY_POS=0.7, VEL_DECAY_SIZE=0.7 per frame (~3.3× предел смещения). Агрессивное затухание предотвращает «улетание» предсказанных боксов в пустоту при панорамировании камеры
- **EMA-сглаживание**: α=0.55 на update, α=1.0 на predict (без лага, Kalman уже сглажен через velocity EMA)
- **maxLost**: по умолчанию 40 кадров (~1.3с при 30fps). Расширен до 300 (~10с) при включённом отслеживании тела — позиция лица от pose держит маску во время окклюзии

### Отслеживание тела — YOLOv8-pose (v0.7.3)

Заменяет запланированный ArcFace re-ID на прагматичный fallback через ключевые точки тела:

- **Модель:** YOLOv8-pose (`yolo26m-pose.onnx`, 82 МБ, AGPL-3.0) — 17 ключевых точек COCO (нос, глаза, плечи, локти, запястья, бёдра, колени, лодыжки)
- **Вход:** 576×704 letterbox, RGB нормализация в [0,1], NCHW-тензор
- **Выход:** `[1, 56, 8400]` — end2end декодирован (cx,cy,w,h,conf + 17×3 ключевых точек в пикселях входного изображения). NMS с IoU=0.5
- **Когда запускается:** на face-detection keyframe'ах, параллельно `detectFaces()` через `Promise.all()`. Троттлинг в Accurate-режиме (каждый 5-й face-keyframe — тела не двигаются покадрово)
- **Сопоставление лиц и тел:** жадный 1-к-1 мин-дистанс на каждом keyframe — исключает привязку двух рядом стоящих людей к одному телу
- **Синтетическая детекция из позы:** свежие позы создают синтетические FaceBox-детекции для тел, чей нос не покрыт ни одной реальной детекцией. Частично видимые лица (край кадра, перекрытие) маскируются сразу — без ожидания следующего keyframe
- **Оверрайд потерянного лица:** когда `framesSinceUpdate > 0`, `faceBoxFromPose()` оценивает позицию лица по носу/глазам и заменяет Kalman-предсказание
- **Оверрайд слабой детекции:** срабатывает также когда площадь лица от pose превышает площадь трека в 1.5× — ловит случай «детектор сжал бокс на повёрнутом лице, но продолжает матчиться»
- **Подавление фантомных масок:** треки с `framesSinceUpdate > 1` И без привязки к телу пропускаются — предотвращает «улетание» Kalman-экстраполированных масок в пустоту при панорамировании камеры
- **EMA-сглаживание:** body-derived face box сглаживается между кадрами (α=0.4) для устранения джиттера ключевых точек
- **Стабильная ширина эффекта:** per-track EMA наблюдаемой ширины лица — растёт мгновенно, убывает с decay 0.99/кадр (~70 кадров полураспад). Предотвращает сжатие ядра пикселизации/размытия при повороте головы
- **Каскад оценки лица `faceBoxFromPose`:** расстояние глаз × 3.0 → расстояние ушей × 1.6 → ширина плеч / 3.0 → |один_глаз − нос| × 6 → |одно_ухо − нос| × 2 → |плечо.y − нос.y| / 1.6 → body bbox / 3.5 → запасной. Односторонние варианты обрабатывают частичный/краевой вид, где видна только одна сторона тела

### Scale-Invariant эффекты

- **Масштабирование силы эффекта:** clamped-линейное `scaleFactor = max(1, faceWidth / 100)` — постоянное количество блоков пикселизации на лицо ≥100px. Маленькие лица сохраняют значение слайдера (factor ≥1).
- **Per-track стабильная ширина:** EMA с мгновенным ростом / медленным спадом — ядро не дёргается при флуктуациях детекторного bbox из-за поворота головы
- **Отступ и feather:** линейно масштабируются с шириной лица (`pad = userValue × faceWidth/100`)
- **Размер эмодзи:** `faceWidth × 0.6` — адаптация к размеру лица на каждом кадре

### Аудио passthrough (v0.7.2)

- **Демукс:** `mediabunny` разбирает MP4/WebM на видео- и аудио-дорожки
- **Ремукс:** `mediabunny` `EncodedAudioPacketSource` — аудио passthrough без перекодирования
- **Pre-roll:** пакеты с отрицательным timestamp (задержка AAC-энкодера) детектятся и пропускаются — MP4-муксер их отвергает
- **Graceful degradation:** аудио passthrough обёрнут в try/catch — при ошибке выходное видео без звука с console.warn, вместо падения всего V2-пайплайна
- **Метаданные:** первый пакет несёт `{ decoderConfig }` от `audioTrack.getDecoderConfig()`. Null decoderConfig обрабатывается с `console.warn` + пропуск аудио
- **Нет аудиодорожки?** Пропускаем, выходное видео без звука

### Прогресс и ETA

- **Прогресс по timestamp:** progress = `sample.timestamp / videoDuration` — точен независимо от переменного FPS или backpressure энкодера
- **Инкрементальные обновления пост-обработки:** encoder.flush (96%), видео passthrough (97%), аудио passthrough (98%), finalize (99%)
- **Fallback offset:** при падении V2 прогресс fallback'а продолжается с последнего значения V2 (без сброса в 0%)
- **ETA:** `remainingDuration / progressRate` — на основе темпорального прогресса, не счётчика кадров

### Память и backpressure (реализовано)

- **`VideoSample.close()`** в try/finally — без утечек GPU-памяти
- **Streaming pipeline:** decode → track → effect → encode — один кадр в полёте, буфер EncodedVideoChunk для muxing
- **Encoder flush:** `encoder.flush()` перед finalize собирает оставшиеся закодированные чанки
- **Отмена:** `AbortController` + проверка `signal.aborted` во всех циклах
- **Body tracking GC:** `lastBodyBoxes` и `trackEffectWidths` очищаются при выбывании треков

### Инфраструктура воркера инференса

- **Централизованный singleton:** `inferenceClient.ts` — один общий воркер для face detection + pose estimation. Заменяет per-pipeline управление воркерами.
- **WebGPU run сериализация:** FIFO-очередь в `inference.worker.ts` — предотвращает ошибки «Session mismatch» при одновременном запуске face detection и pose estimation на одном ORT WebGPU-устройстве
- **Переиспользование сессий:** одинаковый URL модели → одна сессия. И загрузка модели, и создание сессии — ленивые при первом инференсе.
- **NCHW-лейаут:** `preferNchw` параметр в `initSession` — для ESRGAN-моделей использует `{ name: 'webgpu', preferredLayout: 'NCHW' }` + `graphOptimizationLevel: 'basic'` вместо сломанного NHWC
- **Дополнительные входы:** `runMulti` поддерживает `extraInputs` (Float32/Int32/Int64 массивы) — для моделей с несколькими входами (например, baked-in NMS thresholds)
- **Логирование:** `ort.env.logLevel = 'error'` — подавляет per-call warnings (динамические формы вывода, fallback опов), логируются имена входов/выходов
- **Уничтожение воркера:** `terminateInferenceWorker()` — пересоздаёт воркер между бенчмарк-моделями, чтобы избежать интерференции WebGPU-сессий
- **По умолчанию SCRFD-10G** для анонимизации (был SCRFD-500M) — выше полнота детекции, ceil_mode=0 патч под WebGPU
- **Авто-детекция crossOriginIsolated:** `setupRuntime()` проверяет `self.crossOriginIsolated` — если false (GitHub Pages, нет COOP/COEP), выставляет `numThreads=1`. Без этого многопоточный WASM ломает `initWasm()` навсегда.
