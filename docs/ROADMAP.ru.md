## v0.7 — Анонимизация лиц в видео

> Расширяет пайплайн v0.6 Hide Faces на видео. Те же 4 эффекта,
> те же 5 детекторов, но покадрово с трекингом лиц и перекодированием в видеофайл.

### Архитектура

```
MP4/WebM → Canvas-декодинг → Кадр → Детекция (ключевые кадры) → Трекинг (Kalman+IoU)
                                     → [YOLO-pose якорь по телу]
                                     → Эффект → Кодирование → Выходное видео
```

| Слой | Модуль | Назначение |
|------|--------|------------|
| Вход | `VideoAnonymizeWizard.tsx` | Загрузка видео, превью кадра, настройка эффектов, прогресс |
| Декодинг | `videoDecoder.ts` | `<video>` → Canvas → `ImageData` на каждый кадр |
| Детекция | переисп. `anonymize.ts` | Полная детекция каждые N ключевых кадров |
| Трекинг | `faceTracker.ts` | Kalman-фильтр + IoU-матчинг для промежуточных кадров |
| Поза | `poseDetector.ts` (yolo26m-pose) | Ключевые точки тела как якорь для лица, снижает дрейф |
| Эффект | переисп. `anonymizeEffects.ts` | Размытие / Пикселизация / Заливка / Эмодзи на лицо на кадр |
| Кодирование | `videoEncoder.ts` | WebCodecs (приоритет) или ffmpeg.wasm → MP4/WebM |
| Состояние | `videoAnonymizeStore.ts` | Боксы по кадрам, ID треков, прогресс обработки |

### Трекинг лиц (v0.7.1)

- **Интервал ключевых кадров**: полная детекция каждые 10–15 кадров
- **Kalman-фильтр**: 4 состояния (x, y, dx, dy) на лицо, модель постоянной скорости
- **IoU-матчинг**: жадный венгерский алгоритм между предсказанными и детектированными боксами
- **Восстановление дрейфа**: пере-детекция при IoU < 0.3 в течение 3 кадров подряд
- **Новые лица**: любая несматченная детекция → создание трекера

### Интеграция YOLO Pose (v0.7.2)

- Модель: `yolo26m-pose.onnx` (~26 МБ, 17 COCO-точек)
- Каждый человек: bbox + ключевые точки (нос, глаза, уши, плечи, локти, запястья, бёдра, колени, лодыжки)
- Якорь лица относительно тела через нос+глаза → bbox тела; стабилизация при окклюзиях/поворотах
- Только на ключевых кадрах (каждые 10–15, тот же график что и детекция)
- Опционально: пропустить face-детектор, использовать нос+глаза как прокси лица (быстрее, но менее точно)

### Стратегия кодирования

| Метод | Плюсы | Минусы |
|-------|-------|--------|
| **WebCodecs** | GPU-ускорение, без WASM | Chrome/Edge, нет в Firefox |
| **ffmpeg.wasm** | Кроссплатформенность, зрелость | 30+ МБ WASM, CPU, медленно |
| **Гибрид** | WebCodecs если доступен, fallback на ffmpeg.wasm | Дополнительная сложность сборки |

Выбор: WebCodecs первым (Chrome/Edge, 93% пользователей), ffmpeg.wasm как запасной.

### UI

1. Загрузка видео → первый кадр + длительность/разрешение
2. Детекция на кадре 0 → оверлей (переисп. FaceOverlay)
3. Настройка эффектов (переисп. панель эффектов AnonymizeWizard)
4. «Обработать» → прогресс: текущий кадр / всего, ETA, кнопка отмены
5. «Скачать» → выходное видео (WebM VP9, то же разрешение/fps что и вход)
6. До/После: два видеоплеера рядом (или сравнение кадров для выбранного кадра)

- `yolo26m-pose.onnx` → `src/ml/models/` + запись в modelRegistry
- `@ffmpeg/ffmpeg` или `@ffmpeg/util` (если WebCodecs недоступен)
- Без новых runtime-зависимостей в остальном — Canvas API + существующий ORT

## v0.6 — Анонимизация лиц (Hide Faces)

> Двухэтапный воркфлоу: сначала автоматический поиск всех лиц с возможностью ручной коррекции,
> затем применение эффекта (размытие / пикселизация / заливка / эмодзи / стикер).
